Actividad Fase 1

Análisis del problema

Requisitos funcionales

  1. La página web debe contar con las secciones indicadas por el cliente para ofrecer sus servicios y ofertas de empleo de manera pública.
  2. El sistema debe permitir el acceso a una sección privada únicamente para personal autorizado, correspondiente a los procesos internos de la firma.
  3. La página o interfaz privada que supervisarán los integrantes de la firma debe contar con un registro e inicio de sesión.
  4. El sistema debe permitir a los usuarios consultar el estado de evaluación de su hoja de vida.
  5. El sistema debe validar los datos del formulario para evitar el envío de campos vacíos o información duplicada.
  6. La sección de la página encargada de los procesos debe permitir operaciones CRUD (creación, lectura, escritura y eliminación).

Requisitos no funcionales

  1. La página web debe contar con un tiempo de carga inferior a 5 segundos bajo condiciones normales de conexión.
  2. El acceso a la base de datos debe estar restringido únicamente a personal autorizado, mediante un sistema de control de permisos.
  3. El sistema debe mostrar mensajes claros y comprensibles ante errores o tras la realización de cambios en los datos.
  4. El sistema debe cumplir con lineamientos de seguridad, incluyendo el uso de contraseñas seguras.
  5. La interfaz de la página debe ser responsiva.
  6. El sistema debe ofrecer una experiencia de usuario intuitiva y accesible, garantizando una buena usabilidad.
  7. Se deben implementar bases de datos escalables, que soporten un crecimiento en la cantidad de información sin afectar el rendimiento.
  8. El acceso al entorno privado de la empresa debe estar protegido mediante autenticación segura para personal autorizado.

Actores

  • Integrantes de la firma: Personas que ofrecen sus servicios a través de la plataforma y que cuentan con autorización para acceder al entorno privado de la empresa. Interactúan en un apartado interno donde realizan el seguimiento de sus procesos.
  • Administrador de bases de datos: Persona encargada de la administración de las bases de datos, con permisos especiales para gestionar la información y realizar copias de seguridad.
  • Postulantes: Personas interesadas en unirse a la firma, que ingresan a la página para subir su hoja de vida y recibir la evaluación correspondiente. Realizan el formulario de postulación y pueden consultar el estado de su proceso.
  • Evaluadores: Profesionales encargados de recibir, revisar y aprobar o rechazar las hojas de vida enviadas por los postulantes.
  • Clientes: Usuarios que visitan la página para obtener información sobre los servicios legales ofrecidos. Pueden acceder a secciones como profesionales de la firma, trayectoria, contacto, ubicaciones y noticias de interés general a través de una interfaz amigable.

Información a registrar

Interfaz pública

  • Página de inicio: Información introductoria sobre la firma y los servicios ofrecidos.
  • Página de trayectoria: Historia de la firma, experiencia, logros y casos de éxito.
  • Página de noticias: Actualizaciones y artículos de interés relacionados con la firma o el mundo legal.
  • Página de profesionales: Perfiles de los miembros de la firma, incluyendo especialidades y contactos.
  • Página de contacto: Formulario para contactar a la firma, dirección física, teléfono y medios de comunicación.
  • Página de postulación: Formulario para que los postulantes envíen sus hojas de vida y datos para su evaluación. Sección para revisar estado de postulación.

Interfaz privada

  • Página de inicio de sesión: Inicio de sesión para que los usuarios autorizados ingresen al sistema.
  • Página de registro: Formulario para registrar nuevos usuarios (evaluadores, administradores, abogados, etc.).
  • Página de gestión de procesos: Vista para que los miembros internos gestionen los casos y procesos legales.
  • Página para evaluar hojas de vida: Vista para que los evaluadores revisen las hojas de vida de los postulantes y registren su evaluación (aceptado/rechazado).

Frameworks

Actualmente los frameworks son muy utilizados porque las empresas prefieren acelerar la creación de páginas web en lugar de diseñar todo desde cero. Los frameworks ayudan a optimizar el trabajo al ofrecer clases predefinidas, funciones listas para usar y diseños prediseñados que facilitan crear sitios web con buena usabilidad, responsividad y una excelente experiencia de usuario.

Un framework o marco de trabajo es un conjunto de librerías y herramientas que proporcionan una estructura para organizar mejor el código, reutilizarlo fácilmente y lograr diseños responsivos de forma ágil. Existen frameworks especializados tanto para backend como para frontend.

Frameworks para diseño web responsivo y usabilidad

Bootstrap

Es un framework CSS enfocado en un enfoque mobile-first. Ofrece clases utilitarias y componentes UI predefinidos, junto con un sistema de rejilla basado en columnas que facilitan la creación de diseños responsivos. Destaca por su facilidad de uso y su capacidad para crear sitios adaptables de manera sencilla.

Tailwind

Este framework CSS se destaca por su alto nivel de personalización. Funciona con utilidades (clases pequeñas) que permiten construir diseños personalizados directamente en el código HTML. Es liviano y ofrece un buen soporte para crear interfaces responsivas y modernas.

Bulma

Es un framework gratuito y fácil de aprender. Usa Flexbox para crear layouts flexibles y responsivos, facilitando la organización de filas, columnas y el centrado de contenido. También incluye componentes predefinidos como botones y formularios, que mejoran la usabilidad y la interacción del usuario.

Foundation

Foundation es un framework que también prioriza el diseño para dispositivos móviles, lo cual es útil dado el gran uso de estos dispositivos. Además de su sistema de rejilla flexible y sus componentes UI, incluye plantillas prediseñadas para acelerar el desarrollo de páginas web.
